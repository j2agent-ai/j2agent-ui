# 图表渲染

## Mermaid

### 流程

1. `normalizeMermaidSource` 规范化源码
2. `mermaid.parse` + `mermaid.render` 得到 SVG
3. 注入 `.md-diagram-body`，`scheduleFitDiagramSvg` 适配尺寸
4. xychart 块额外添加 class `md-diagram-xychart`

### xychart-beta 与 TOPN 标签重叠

**现象**：竖向柱状图在聊天气泡较窄时，X 轴站点名挤在一起（如「当前告警站点分布 TOP15」）。

**原因**：

1. Mermaid 竖向 `xychart` 对多类目、长文本无旋转/截断（上游 issue #5926 / #7599）
2. `fitDiagramSvgInBody` 曾按宽度等比缩小整张 SVG，文字不重排，叠字更明显

**前端修正**（`markdownRenderer.ts`）：

| 机制 | 条件 | 行为 |
|------|------|------|
| `normalizeXychartOrientation` | `xychart` / `xychart-beta`，`x-axis [...]` 类目 ≥ 8，且无 `horizontal` | 首行改为 `xychart-beta horizontal` |
| 宽图模式 | SVG 自然宽度 > 气泡内容区宽度 | 不压窄宽度；`md-diagram-body--wide` + 横向滚动 |

类目标签由 Mermaid 默认横排；**竖向图**（未写 `horizontal`）在渲染后对底部 `g.bottom-axis` 标签做超宽换行（SVG `tspan`），宽度按相邻刻度间距估算；换行后同步扩展 SVG `viewBox` 与气泡容器高度，避免底部被裁切。
| 高度适配 | 始终 | 仍受 `--md-diagram-max-height` 限制 |

Agent 侧推荐在 Skill 中直接写 `xychart-beta horizontal`（类目 ≥ 8），减少依赖自动修正。

### 柱/条颜色（横竖向一致）

- Mermaid 初始化时注入 `themeVariables.xyChart.plotColorPalette`，与饼图 `pie1`–`pie8` 同色板（首色 `#93C5FD`）。
- 渲染后对 `g.bar-plot-* rect` 执行 `applyXychartBarColors`，避免竖向图仍走默认紫系、与横向不一致。
- CSS 变量 `--n-color-diagram-bar` 作兜底。

### 其它 Mermaid 类型

- **pie**：`quotePieSliceLines` 为未加引号的扇区标签补双引号
- **flowchart / sequenceDiagram**：依赖主题变量，无额外缩放逻辑

### 错误展示

解析或渲染失败时，块加 `md-diagram-error`，body 内为标题 + `<details>` 错误详情，避免被 `overflow: hidden` 裁切。

## Vega-Lite

- 围栏：`vega-lite` / `vegalite`
- `parseVegaLiteSpec`：JSON 规范化 + `enhanceVegaLiteSpec`
- **多类目 X 轴**：`encoding.x` 为 nominal/ordinal，`data.values` 去重类目 ≥ 8，且未配置 `axis.labelAngle` 时，自动 `labelAngle: -45`、`labelLimit: 120`
- 嵌入：`vega-embed`，`renderer: 'svg'`，`actions: false`

## PlantUML

- 静态脚本：`/vendor/plantuml-js/plantuml.js`
- 输出 SVG 注入 body，走同一套 `fitDiagramSvgInBody`

## 尺寸与 ResizeObserver

- CSS 变量 `--md-diagram-max-height`（默认 `min(360px, 42vh)`，见 `markdown.scss`）
- `fitDiagramSvgInBody` 读取 viewBox / width / height / getBBox 作为自然尺寸
- `ResizeObserver` 监听气泡、图表块宽度变化；`window.resize` 时失效高度缓存并重算

## 全屏预览

`ChatView.vue`：点击 `.md-diagram-body svg`（非 error 块）收集同气泡内所有 SVG，转 blob URL 后用 `ElImageViewer` 切换预览。

## 为什么改了代码，旧会话里的图「刷新」不变？

**不是 Mermaid 把 SVG 缓存在磁盘**，也**不是**后端把图表存进数据库。消息里只有 Markdown 原文，每次展示都会走 `renderMarkdown` + `renderMarkdownBlocks`。

常见原因有两类：

| 原因 | 说明 | 处理 |
|------|------|------|
| **浏览器 / 部署仍是旧前端包** | 生产环境静态资源在 `${AI_CENTER_BASE_PATH}/ui`，带 hash 的 `assets/*.js` 会被浏览器长期缓存；若未重新 `npm run build` 并同步 `dist`，或只普通刷新，仍执行旧 JS | 重新构建并发布 `dist`；浏览器 **强制刷新**（Cmd/Ctrl+Shift+R）或无痕窗口；见 [前端静态资源更新](../../../inc-backend/j2agent/docs/docker部署/前端静态资源更新.md) |
| **SPA 内 DOM 未重建** | `v-html` 输出的 HTML 字符串若与上次相同，Vue 可能不替换已有 DOM；图表块上已有 `data-md-rendered="true"` 时，`renderMarkdownBlocks` 会直接跳过 | 已用 `MARKDOWN_RENDERER_REVISION` + 气泡 `:key`：前端版本递增后，历史消息会强制重绘图表 |

开发环境请确认 `npm run dev` 已热更新；若仍像旧版，重启 dev 并硬刷新页面。
